# Apartment-Management-System

Project Objectives
- Simplify apartment and tenant management
- Digitize tenant registration and flat selection
- Improve transparency in flat availability
- Provide a clean and easy-to-use web interface
- Apply real-world software engineering practices

 Technology Stack
- **Frontend:** HTML, CSS, JavaScript  
- **Backend:** PHP  
- **Database:** MySQL (via PHP)  
- **Design Assets:** CSS styling and apartment images  
- **Architecture:** Data Flow Diagram (DFD)

Application Workflow:

1️⃣ Home Page
- Acts as the entry point to the application
- Displays apartment-related visuals and navigation options

2️⃣ User Registration
- New users can register using the **registration module**
- User details are validated and stored in the database

3️⃣ User Login
- Registered users can log in using credentials
- Authentication handled using PHP
- Error handling implemented for invalid login attempts

4️⃣ Flat Selection
- Users can view available apartment options
- Selection process helps track apartment availability

Key Features
- Tenant registration and login
- Flat availability selection
- Form validation and error handling
- Clean UI with responsive styling
- Modular PHP backend structure

### 5️⃣ Backend Processing
- PHP scripts handle form submissions and validations
- Errors are managed through a centralized error-handling file
